Preparation

Field Panel Connection Check

  • Turn off and close ZScaler
  • Disconnect from ABT Site
  • Confirm the computer IP address and subnet mask are on the same subnet intended for the field panel
  • Set up BacStac by going to C:\ProgramFiles(x86)\Cimetrics and running bacstac.bat
  • Set up a physical connection if one does not exist
  • Select the ethernet adaptor used to connect to the panel (the computer IP should autopopulate in the dropdown
  • Select "Attached"
  • Bacstac will be restarted automatically
  • Unzip the CT database in your Job Folder
  • Open Job Editor
  • Open Online
  • Select the appropriate atom.fdb file for your job
  • Open System Profile
  • Insert new Insight
  • Name must match Computer name. Computer name can be found by opening Control Panel on Windows and Clicking on System
  • Move the BLNs from the existing Insight into your Insight
  • Confirm the three names of the field panel (Name, System Name, Object Name), the panel instance ID, and the field panel IP Address ensuring that it is on the same subnet as your computer
  • Open MMI database transfer
  • Open your BLN on the left
  • Click the panel on the right
  • Direct Connect
  • Open Terminal Emulation via the "Tools" Menu
  • Enter the initial login and password ("high" for both) and change the password
  • Enter the characters 'S','H','E','S','M'
  • Enter the correct field panel name and IP address of the panel being mindful that the IP address of the field panel should be on the same subnet as the computer
  • Close CommTool completely
  • Open the job Online via Job Editor
  • Open System Profile
  • Panel should be connected

Air System Basic Hardware Check

  • Using a magnehelic pressure guage and hand pump, ensure that all low and high static cutouts are tuned appropriately where low-static cutouts are typically tuned to -4"wc and high static cutouts at 4"wc
  • If the unit is equipped with a high-humidity cutout sensor, ensure the humidity sensor and cutout are an adequate distance from the humidifier to ensure they are not being sprayed directly
  • Visually confirm all Modulating Dampers and Actuators accept 0%, 50%, 100% commands where 0% is fully closed and 100% is fully open. If Feedback Signals are used, confirm feedback signals at the front end correspond to what you see visually.
  • Visually Confirm all Two-Position Dampers and Actuators accept 'Off' and 'On' commands where 'Off' is fully closed and 'On' is fully open. If Feedback Signals are used, confirm feedback signals at the front end correspond to what you see visually.
  • Release all dampers and actuators from Operator Priority.
  • If the unit is equipped with mixing dampers, confirm that economizer, exhaust and return dampers all modulate based on the 'MAD' or 'RAD' command where the mixing (return) damper actuates the opposite of the others
  • Release all dampers from Operator Priority.
  • Command open the dampers to allow airflow to be drawn
  • Confirm all Fans go to minimum speed when started, and accept half & full speed commands from the BMS as Hz or %
  • Confirm all Fan Proof points turn on when Fans are commanded 'On' with a 0 Hz or % speed command
  • Confirm all Fans go to 0% or 0Hz when commanded to stop
  • Confirm all Fan proof points turn off when fans are commanded 'Off'
  • Release all dampers and fan commands from Operator priority

Air System Return From Power Loss

  • Locate On Power Return "ONPWRT" section
  • Confirm all On/Off commands are turned off, Fan Speeds are commanded to 0 Hz or %, all Dampers and Actuators are commanded to 0% or 'Off', and all timers and counters are set to 0

  • Confirm the Lead Isolation Damper opens
  • Confirm the Lead Fan starts upon Open Proof of the Lead Isolation Damper
  • Confirm Fan(s) and corresponding Isolation Damper(s) open together
  • Confirm Outside Air Damper opens
  • Confirm Supply Fan starts upon Open Proof of Outdoor Air Damper
  • Confirm the discharge flow setpoint and return flow setpoint match one another
  • Confirm supply and return fans start together
  • Confirm supply and return fans start together
  • Confirm the Supply Fan Starts
  • Confirm the mixing damper is closed prior to the Return Fan Starting
  • Confirm the mixing damper is only allowed to open upon Proof of Return Fan running
  • For each high static and low static sensor, mimic the condition by either disconnecting or shorting the alarm wires at the field panel
  • Confirm all Fans turn off, and all Speeds, Dampers, Actuators, and Timers are either turned commanded to 0 or their relinquish default positions
  • Confirm all lines for Air System Disable are traced and all lines for Air System Enable are skipped
  • Reconnect the wires at the field panel
  • Confirm all lines for Air System Enable are traced and all lines for Air System Disable are skipped

Fan Wall Lead/Lag + Weekly Turnover

  • Command all fan speeds to 20%
  • Command the Loop Output that commands fan speeds to 100%
  • Confirm that after 5 minutes, and additional fan is called to run if all fans are not already running
  • Confirm that the next fan is commanded on, and that the isolation damper is only commanded open after the fan proofs on. This will prevent an overcurrent alarm
  • Confirm the Fan Speed Ramp Timer slowly increases
  • Command all fan speeds to 20%
  • Command the Loop Output that commands fan speeds to 30%
  • Confirm that after 5 minutes, 1 fan is called to turn off if at least 2 fans are running
  • Confirm the Fan is commanded off and Isolation Damper is commanded closed
  • Open each Fan Start/Stop point with "Commander" and reset totalization
  • Set the weekly turnover to happen 5 minutes from now
  • When the changeover happens, confirm the following takes place:
  • 1. All fans currently running remain running
  • 2. If available and not already running, the new lead fan is commanded on. This fan should be the one with the lowest runtime
  • 3. The fan that was the Lead previously is now the lag
  • 4. Confirm that after 5 minutes, the previous Lead Fan is called to turn off
  • Release the Loop Output point controlling Fan Speeds from Operator Priority
  • Release the Fan Speed Commands from Operator Priority
  • Set the weekly turnover to happen at the day and time specified in the Sequence of Operation

Air System Stop Heating Mode

  • Confirm the "ZONE.MODE" point is currently 2 or Occupied
  • Open the Bacnet Schedule and Edit the SSTO (Schedule Optimization" parameters
  • Confirm "Stop Optimization" is checked
  • Go to the "Stop" tab and confirm the Heating Mode is set for "STOP_HTG"
  • Record the current "Max Stop Duration" and Adjust it to 5 minutes
  • Record the current "Calculation Constant" and Adjust it to 1 minute
  • Record the Differential Temperature (Default = 1.8 degF)
  • Confirm "Use Learning" is unchecked
  • Command the Stop Heating Setpoint to the value of the current "Inside Temperature" used for SSTO - the Differential Temperature from the previous step + 0.5
  • Press "OK"
  • Change the Bacnet Schedule to end Occupancy in 10 minutes
  • Wait 5-6 minutes and confirm "ZONE.MODE" is commanded to the Stop-Heating Mode Value
  • Confirm all lines for Stop-Heating are traced, and the Air System turns off to conserve energy
  • Wait 4-5 minutes and confirm the "ZONE.MODE" gets commanded to Vacancy (Unoccupied) Mode
  • Release the Stop Heating Setpoint from Operator Priority
  • Adjust the Max Stop Duration back to the recorded value
  • Adjust the Calculation Constant back to the recorded value
  • Confirm "Use Learning" is checked
  • Adjust the Bacnet Schedule Occupancy end time back to its original value

Air System Warmup Mode

  • Confirm the "ZONE.MODE" point is currently 1 or Vacancy (Unoccupied)
  • Open the Bacnet Schedule and Edit the SSTO (Schedule Optimization" parameters
  • Confirm "Start Optimization" is checked
  • Confirm the "Allowed Operation" and "Desired Operation" are both set to "Heating" or "Heating and Cooling"
  • Go to the "Start" tab and confirm the "Heating Mode" is set to "WARMUP"
  • Confirm "If too early" is set to "Switch to next scheduled occupancy mode"
  • Confirm "If too late" is set to "Switch to mode WARMUP"
  • Uncheck Use Learning and click "Advanced"
  • Set the "Outside Temp Limit" to a value higher than the current Outside Air Temperature. If the current Outside Air Temperature is > 87.8 degF leave the "Outside Temp Limit" at 59 degF
  • Record the "Inside Temp Deviation" value
  • Set the "Min Start Duration" to 0
  • Set the "Max Start Duration" to 10
  • Set the "Max Extension Time" to 0
  • Set the "Max Setpoint Offset" to 0
  • Press "OK"
  • Go to the "General" tab
  • If the current Outside Air Temperature > 87.8 degF change "Outside Temperature" from a point name to numerical value and set the value to 45 degF
  • Command the "Heating Set Points - During Occupancy" to the current "Inside Temperature" - "Inside Temp Deviation" - 1
  • Press "OK"
  • Set the Occupancy Start Time to 12 minutes from now. The Occupancy End Time should be adjusted accordingly
  • Press "OK"
  • Wait 2-3 minutes and confirm "ZONE.MODE" is commanded to the Morning-Warmup Mode Value
  • Confirm all lines for Morning Warmup are traced, and the Air System turns on to control airflow/static and temperature to bring the space up to the Occupied Heating Setpoint in preparation for occupancy
  • Confirm that after 10 minutes of the Air System being in Warmup Mode, the "ZONE.MODE" point gets commanded to Occupied Mode
  • Release the Occupied Heating Setpoint from Operator Priority
  • Open the Bacnet Schedule and Edit the SSTO (Schedule Optimization" parameters
  • If the Outside Air Temperature reference was changed to a numerical value, change it back to the Outside Air Temperature point that should be referenced
  • Open the "Start" tab and check the "Use Learning" checkbox
  • Click on the "Advanced" button and set the "Outside Temp Limit" to its recorded value (Default is 59 degF)
  • Set the "Max Start Duration" to 60 minutes
  • Press "OK"
  • Press "OK"
  • Adjust the Occupied Start & End Times back to their normal schedule
