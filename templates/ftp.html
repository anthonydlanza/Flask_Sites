Field Panel Connection Check

  • Turn off and close ZScaler
  • Disconnect from ABT Site
  • Confirm the computer IP address and subnet mask are on the same subnet intended for the field panel
  • Set up BacStac by going to C:\ProgramFiles(x86)\Cimetrics and running bacstac.bat
  • Set up a physical connection if one does not exist
  • Select the ethernet adaptor used to connect to the panel (the computer IP should autopopulate in the dropdown
  • Select "Attached"
  • Bacstac will be restarted automatically
  • Unzip the CT database in your Job Folder
  • Open Job Editor
  • Open Online
  • Select the appropriate atom.fdb file for your job
  • Open System Profile
  • Insert new Insight
  • Name must match Computer name. Computer name can be found by opening Control Panel on Windows and Clicking on System
  • Move the BLNs from the existing Insight into your Insight
  • Confirm the three names of the field panel (Name, System Name, Object Name), the panel instance ID, and the field panel IP Address ensuring that it is on the same subnet as your computer
  • Open MMI database transfer
  • Open your BLN on the left
  • Click the panel on the right
  • Direct Connect
  • Open Terminal Emulation via the "Tools" Menu
  • Enter the initial login and password ("high" for both) and change the password
  • Enter the characters 'S','H','E','S','M'
  • Enter the correct field panel name and IP address of the panel being mindful that the IP address of the field panel should be on the same subnet as the computer
  • Close CommTool completely
  • Open the job Online via Job Editor
  • Open System Profile
  • Panel should be connected

DXR Connection Check

  • Turn off and close ZScaler
  • Open Services (Windows)
  • Stop Bacstac Protocol
  • Confirm the computer IP address and subnet mask are on the same subnet intended for the dxrs (Ethernet DXRs only)
  • Unzip the ABT Site database in your Job Folder
  • Open ABT Site
  • Select the Project you're working on
  • Default Username: Administrator
  • Default Password: OneBT.123
  • Click on the Start-up Tab
  • Open the Connection Settings in the top-right corner
  • Choose the method of connection (Room unit connection, USB, Ethernet, Cloud)
  • Choose the correct Target Selection
  • Choose the network interface you will be using to interface with the dxr(s)
  • Confirm the "Include discovery in Startup" checkbox is selected
  • Click "Connect"
  • Confirm your DXRs begin to populate under "Discovered Devices"

Terminal Equipment Basic Hardware Check

  • Log into the DXR SSA (Online View) and navigate to the Room Segment Screen
  • Confirm Pressure tubes are hooked up correctly with the High Pressure Tube hooked up to P+ (left) and the Low Pressure Tube hooked up to P- (right)
  • Confirm All Relay Contacts and Damper/Valve Actuators are hooked up correctly at the DXR and Control Device
  • For Three-Point Floating Actuators, There should be 3 wires -- 24VAC to Open, Common, 24VAC to Close
  • For 0-10V Actuators, There should be 3 wires -- 24VAC Power, Common, 10VDC signal
  • Relay Contacts will need 2 wires in most instances -- 24VAC Power, Common. Some devices may only need 24VAC
  • For each Damper and Actuator Complete the following sequence:
  • Command the Actuator to 0% and visually inspect the actuator to confirm it has closed completly and holds its position
  • Command the Actuator to 100% and visually inspect the actuator to confirm it has opened completly and holds its position
  • Release the point from Override (the orange hand should be gone) and confirm normal operation
  • For each solenoid valve and relay contact, confirm the following sequence takes place
  • Navigate to the point properties (three dots on the right)
  • Confirm the "End position synchronization" is set to "Continuous"
  • Return to the previous page
  • Command the point "On" and visually confirm that the solenoid valve or relay has turned on
  • Command the point "Off" and visually confirm that the solenoid valve or relay has turned off
  • Release the point from Override (the orange hand should be gone) and confirm normal operation
  • Confirm all sensors and safety contacts are hooked up correctly at both the DXR and the device
  • For Resistance-Based sensors (temperature) there will be 2 wires -- Signal, Common
  • For Voltage-Based sensors there will be 2 wires in most cases -- Signal, Common. Some devices may require 24VAC Power, Common, and Signal
  • For Current-Based sensors on platforms other than DXR2.E17C and DXR2.E17CX, a 24VDC power supply such as the Kele DCP-1.5-W may be needed, along with a 500ohm resistor to convert the 4-20ma signal to 2-10Volt. In this situation 2 wires will be connected at the DXR -- Signal, Common
  • For Current-Based sensors on platforms DXR2.E17C and DXR2.E17CX, loop powered devices will have 2 wires -- 24VDC Power, Signal
  • For Dry-Contacts like Presence Detectors, Window Contacts, and Condensation Monitors, there will be 2 wires -- Signal and Common
  • For each analog sensor, confirm the following sequence takes place:
  • Confirm the sensor registers a normal value at the DXR
  • Disconnect the signal wire at the sensor
  • Confirm the sensor registers as failed at the DXR
  • Reconnect the signal wire at the sensor
  • Confirm the sensor registers a normal value at the DXR
  • Confirm each Room Operator Unit (T-stat) is displaying the correct temperature by comparing that temperature with a reading from a thermometer or temp-gun
  • If an offset needs to be made, navigate to Room-Segment-->Room operator unit device #-->Room temperature properties (three-dots) and adjust the "Correction offset" as necessary
  • Confirm the temperature displayed matches what is being registered at the DXR

Presence Detection Detection

  • For each Presence Detector, confirm the following sequence takes place:
  • While in the Room Segment, Navigate to the point properties (three dots on the right)
  • Confirm the "Present Value" matches what you see happening. For example, if nobody has been in the room, but the present value says "Present", you will need to change the polarity from "Normal" to "Reverse" or vice-versa.
  • Confirm the "Switch-Off" & "Switch-On" Delays are correct
  • Physically enter the room to trigger the presence detector
  • In the DXR online view, navigate to Room-->Room Operating Mode Determination and confirm that "Plant operating mode" is set to Comfort
  • Physically leave the room long enough for the presence detector to register an absence
  • Confirm the "Plant operating mode" is set to either "Pre-Comfort" or "Economy"

Window Contacts

  • For each Window Contact, confirm the following sequence takes place:
  • While in the Room Segment, Navigate to the point properties (three dots on the right)
  • Confirm the "Present Value" matches what you see happening. For example, if the window is physically open, but it is registering as closed, you will need to change the polarity from "Normal" to "Reverse" or vice-versa
  • Physically open the window
  • In the DXR online view, navigate to Room-->Room Operating Mode Determination and confirm that "Plant operating mode" is set to "Protection"
  • Close the window
  • Wait 2-5 minutes and confirm the "Plant operating mode" leaves "Protection"

Condensate/Overtemp Detection

  • For each Condensation or Overtemp Detector, confirm the following sequence takes place:
  • While in the Room Segment, Navigate to the point properties (three dots on the right)
  • Confirm the "Present Value" matches what you see happening. For example, if the pipe is dry, but is registering a condensation alarm, you will need to change the polarity from "Normal" to "Reverse" or vice-versa
  • Short the wires at the device
  • Confirm a change of state at the DXR. For example, you should see two things happen: An alarm should register at the detector, and the associated actuator should be locked out
  • Reconnect the wires at the device
  • Confirm the state of the detector returns to normal and the actuator is released from lockout

Terminal Control

Air System Basic Hardware Check

  • Using a magnehelic pressure guage and hand pump, ensure that all low and high static cutouts are tuned appropriately where low-static cutouts are typically tuned to -4"wc and high static cutouts at 4"wc
  • If the unit is equipped with a high-humidity cutout sensor, ensure the humidity sensor and cutout are an adequate distance from the humidifier to ensure they are not being sprayed directly
  • Visually confirm all Modulating Dampers and Actuators accept 0%, 50%, 100% commands where 0% is fully closed and 100% is fully open. If Feedback Signals are used, confirm feedback signals at the front end correspond to what you see visually.
  • Visually Confirm all Two-Position Dampers and Actuators accept 'Off' and 'On' commands where 'Off' is fully closed and 'On' is fully open. If Feedback Signals are used, confirm feedback signals at the front end correspond to what you see visually.
  • Release all dampers and actuators from Operator Priority.
  • If the unit is equipped with mixing dampers, confirm that economizer, exhaust and return dampers all modulate based on the 'MAD' or 'RAD' command where the mixing (return) damper actuates the opposite of the others
  • Release all dampers from Operator Priority.
  • Command open the dampers to allow airflow to be drawn
  • Confirm all Fans go to minimum speed when started, and accept half & full speed commands from the BMS as Hz or %
  • Confirm all Fan Proof points turn on when Fans are commanded 'On' with a 0 Hz or % speed command
  • Confirm all Fans go to 0% or 0Hz when commanded to stop
  • Confirm all Fan proof points turn off when fans are commanded 'Off'
  • Release all dampers and fan commands from Operator priority

Air System Return From Power Loss

  • Locate On Power Return "ONPWRT" section
  • Confirm all On/Off commands are turned off, Fan Speeds are commanded to 0 Hz or %, all Dampers and Actuators are commanded to 0% or 'Off', and all timers and counters are set to 0

Chilled Water System Enable

  • Confirm "FLOW.SP.DP" for each chiller is set to a value other than 0
  • Command "CHW.ENABLE" to "On"
  • Confirm "CHLR.STGUP.TIME" begins to count up. A resident timer in the program "CHLR.STG.DLY" will count up at the same rate
  • After 30 minutes elapses (as counted by "CHLR.STG.DLY") confirm that "CHLR.REQ" changes to 1 and that the Lead Chiller Isolation Valves begin to open
  • If testing virtually, manually override the Chilled Water and Condensor Water Isolation Valve Feedback Signals ("CH#.CHW.IV.FDBK" & "CH#.CDW.IV.FDBK") to 100% upon the Valve Commands going above 7%
  • If testing virtually, manually override the Condenser Water System Cells Open Count "CT.CELL.OPN" to 1. This point is the sent from the Condenser Water System upon proof open of a cooling towers Supply and Return Isolation Valves
  • Once the Lead Chiller Isolation Valves are proofed open, confirm that the lead Chilled Water and Condensor Water Pumps Start
  • If testing virtually, manually override the Chilled Water and Condensor Water Pump Statuses ("CHWP#.STATUS" & "CDWP#.STATUS") to "On" upon the Pumps turning "On"
  • If testing the Chilled Water System seperately from the Condensor Water System, Command "CHLR.CTR" to "On". This point is the "Clear To Run" signal from the Condensor Water System whenever 1 or more Cooling Tower Fans have been running for more than 30 seconds
  • Confirm the Lead Chiller ("CHLR.ENA") turns "On"
  • If testing virtually, command ("CH#.STATUS") "On"
  • Confirm "CHLR.COUNT" goes to 1
  • Confirm "CHLR.STG.DLY" Restarts and begins counting up again
  • Command the Chilled Water Differential Pressure Setpoint ("MAIN.DP.SP") to 15 psi
  • Confirm that Central Plant Main Bypass Valve ("CHW.BPV") begins to close
  • Command the Chilled Water Differential Pressure Setpoint ("MAIN.DP.SP") to 5 psi
  • Confirm that Central Plant Main Bypass Valve ("CHW.BPV") begins to open
  • Release the Chilled Water Differential Pressure Setpoint ("MAIN.DP.SP") from Operator priority

Chilled Water System Chiller Staging

  • Ensure one of the following three conditions is true to allow "CHLR.STGUP.TIME" to count up:
  • 1. Lead Chiller Full Load Amps (FLA) > 90
  • 2. "CHW.MAIN.DP" < "CHW.MAIN.DP.SP" - 2
  • 3. "CHW.ST" > "CHW.MAIN.TS.SP" + 2
  • After 30 minutes elapses (as counted by "CHLR.STG.DLY") since the start of the first chiller as well as the stage up conditions are met (as indicated by "CHLR.STGUP.TIME") confirm "CHLR.REQ" changes to 2 and that the Lag Chilled Water and Condensor Water Pumps start
  • If testing virtually, manually override the Chilled Water and Condensor Water Pump Statuses ("CHWP#.STATUS" & "CDWP#.STATUS") to "On" upon the Pumps turning "On"
  • In order for the isolation valves on the Lag Chiller to open, the Pump Speeds ("CHWP.SPD" & "CWP.SPD") must not be greater than the Minimum Pump Speeds ("CHWP.MINSPD" & "CWP.MINSPD") - 5%. Confirm the Pump Speeds are within this tolerance
  • Once the Pump Speeds are close to their designated minimums, confirm the Lag Chiller Chilled Water and Condensor Water Isolation Valves begin to open
  • If testing virtually, manually override the Chilled Water and Condensor Water Isolation Valve Feedback Signals ("CH#.CHW.IV.FDBK" & "CH#.CDW.IV.FDBK") to 100% upon the Valve Commands going above 7%
  • Upon the valves proofing open, Confirm the Lag Chiller ("CHLR.ENA") turns "On" and that "CHLR.COUNT" goes to 2
  • If testing virtually, manually override "CH#.STATUS" to "On" once ("CHLR.ENA") turns "On"
  • Confirm "CHLR.COUNT" goes to 2
  • Confirm "CHLR.STG.DLY" Restarts and begins counting up again
  • If testing virtually, command the statuses of one Chilled Water Pump and one Condensor Water Pump that are running ("CHWP#.STATUS"/"CDWP#.STATUS") to "Off"
  • If testing in person, turn off the VFD/Mag Starters for one Chilled Water Pump and one Condensor Water Pump that are running
  • Confirm the Backup Pump (Lag Pump 2) turns "On"
  • If testing virtually, command the Backup Pump Status ("CHWP#.STATUS" / "CDWP#.STATUS") to "On"
  • Confirm the Backup Pump Speed Ramps up to match the speed of the Lead Pump
  • In order to stage down Chillers, you must simulate low-load conditions by dropping the Full Load Amps (FLA) of each chiller...
  • Ensure one of the following three conditions is true to allow "CHLR.STGUP.TIME" to zero out and allow "CHLR.STGDN.TIME" to count up:
  • 1. All Chiller Full Load Amps (FLA) <= 25
  • 2. "CHW.MAIN.DP" > "CHW.MAIN.DP.SP"
  • 3. "CHW.ST" < "CHW.MAIN.TS.SP"
  • Confirm "CHLR.STGUP.TIME" zeros out and that "CHLR.STGDN.TIME" begins to count up. A resident timer in the program "CHLR.STG.DLY" will count up at the same rate
  • After 30 minutes elapses (as counted by "CHLR.STG.DLY") since the start of the second chiller as well as the stage down conditions are met (as indicated by "CHLR.STGDN.TIME") confirm "CHLR.REQ" changes to 1 and that the Lag Chiller ("CHLR.ENA") turns "Off". The Lag Chiller will be the chiller with the highest runtime
  • If testing virtually, manually override "CH#.STATUS" to "Off" once ("CHLR.ENA") turns "Off"
  • Confirm "CHLR.STG.DLY" Restarts and begins counting up again
  • Confirm the Lag Chiller Chilled Water and Condensor Water Isolation valves close after the Lag Chiller has been off for 3 minutes
  • If testing virtually, manually override the Chilled Water and Condensor Water Isolation Valve Feedback Signals ("CH#.CHW.IV.FDBK" & "CH#.CDW.IV.FDBK") to 0% upon the Valve Commands going to 0%
  • If testing virtually, command the statuses of the "failed" pumps from Chiller Staging Test #2 ("CHWP#.STATUS"/"CDWP#.STATUS") to "On"
  • If testing in person, turn on the VFD/Mag Starter for the pumps you turned off in Weekly Rotation Test #2
  • Confirm the Backup Pumps (Chilled Water & Condensor Water Lag Pump 2) turn "Off"
  • If testing virtually, command the Backup Pump Statuses ("CHWP#.STATUS" / "CDWP#.STATUS") to "Off"

Chilled Water System Weekly Rotation

  • If testing virtually, go to the section of the code with the Date and Time for the changeover to occur. Replace the Date and Time arguments with a Test Point you can use for testing
  • If testing virtually, Command your Test Point "On"
  • Confirm "CHLR.ROT" turns on and that "CHLR.REQ" changes to 2
  • Command your Test Point "Off"
  • If testing in person, go to the section of the code with the Date and Time for changeover to occur. Change the Day to today and the time to 2 minutes from now

  • Confirm "If too late" is set to "Switch to mode WARMUP"
  • Uncheck Use Learning and click "Advanced"
  • Set the "Outside Temp Limit" to a value higher than the current Outside Air Temperature. If the current Outside Air Temperature is > 87.8 degF leave the "Outside Temp Limit" at 59 degF

  • Press "OK"
  • Press "OK"
  • Adjust the Occupied Start & End Times back to their normal schedule
